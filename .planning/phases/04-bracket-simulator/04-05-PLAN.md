Modify `src/simulator/simulate.py` to handle the `override_map` parameter in both deterministic and Monte Carlo modes.

**Deterministic mode changes:**
In the slot traversal loop, before resolving teams and calling predict_fn, check if the current slot_id is in override_map:
```python
if override_map and slot_id in override_map:
    slot_winner[slot_id] = override_map[slot_id]
    slot_prob[slot_id] = None  # forced -- no probability
    continue
```
In the output dict, mark overridden slots clearly:
```python
result['slots'][slot_id] = {
    'team_id': int(slot_winner[slot_id]),
    'win_prob': float(slot_prob[slot_id]) if slot_prob[slot_id] is not None else None,
    'round': ROUND_NAMES.get(round_num, f'Round {round_num}'),
    'overridden': slot_id in (override_map or {}),
}
```

**Monte Carlo mode changes:**
Before the slot traversal loop, pre-fill overridden slots:
```python
overridden = set()
if override_map:
    for slot_id, team_id in override_map.items():
        if team_id not in team_to_idx:
            raise ValueError(
                f"Override team {team_id} for slot {slot_id} not in seedings. "
                "Override teams must be tournament participants."
            )
        occupants[slot_id] = np.full(n_runs, team_to_idx[team_id], dtype=np.int32)
        overridden.add(slot_id)
```
In the slot traversal loop, skip overridden slots:
```python
if slot_id in overridden:
    continue  # already pre-filled
```

**Validation:**
- If override_map contains a slot_id not in the slot tree, raise `ValueError(f"Unknown slot {slot_id} in override_map")`.
- If override_map contains a team_id not in seedings values, raise `ValueError` explaining the team must be a tournament participant.
- Validate override_map at the top of simulate_bracket() before either mode branches.
