<objective>
Build the evaluation pipeline that computes Brier score and log-loss per holdout year, compares against chalk baseline, generates a calibration curve, and validates that the model does not produce overconfident predictions.

Purpose: This plan answers the critical question: "Does the baseline model actually beat naive seed-based predictions?" It establishes the quantitative benchmark (Brier < 0.23) that the Phase 6 ensemble must beat, and the calibration check ensures the probabilities are trustworthy for downstream Monte Carlo simulation (Phase 4). The evaluation results become the first row in the Phase 7 model comparison dashboard.

Output:
- `src/models/evaluate.py` — evaluation pipeline with chalk comparison and calibration check
- `models/calibration_curve.png` — reliability diagram
- `models/evaluation_results.json` — machine-readable per-year metrics
</objective>

Create `src/models/evaluate.py` as a **single complete file** containing ALL evaluation functions and the `__main__` block. Do NOT split across tasks — this file is written once, in full.

**Imports:** sklearn.metrics (brier_score_loss, log_loss), sklearn.calibration (CalibrationDisplay, calibration_curve), sklearn.preprocessing.StandardScaler, sklearn.linear_model.LogisticRegression, matplotlib.pyplot, numpy, pandas, json, pathlib

**Function `compute_chalk_brier(y_true)`:**
- Computes hard-chalk Brier score: the Brier score if you always predict P=1.0 that the higher seed (team_a) wins.
- Since label=1 means the higher seed won: `chalk_prob = np.ones(len(y_true))`.
- Return `brier_score_loss(y_true, chalk_prob)`.

**Function `evaluate_all_holdout_years(model_path='models/logistic_baseline.joblib', processed_dir='data/processed')`:**
- Imports `build_matchup_dataset`, `FEATURE_COLS` from `src.models.features`.
- Imports `walk_forward_splits`, `BACKTEST_YEARS` from `src.models.temporal_cv`.
- Imports `load_model` from `src.models.train_logistic`.
- Loads the trained model artifact (model, scaler object from training) — but NOTE: for honest evaluation, we must NOT use the scaler fit on all data. Instead, for each fold:
  1. Fit a NEW StandardScaler on the training fold only.
  2. Transform both train and test with this fold-specific scaler.
  3. Fit a NEW LogisticRegression with the same C parameter (from artifact['best_C']) on the training fold.
  4. Predict probabilities on the test fold.
- This is the correct walk-forward evaluation: model is re-fit per fold using only past data.
- For each holdout year, compute:
  - `brier = brier_score_loss(y_test, y_prob)`
  - `ll = log_loss(y_test, y_prob)`
  - `chalk_brier = compute_chalk_brier(y_test)`
  - `accuracy = (y_pred_class == y_test).mean()` where `y_pred_class = (y_prob >= 0.5).astype(int)`
  - `n_test = len(y_test)`
  - `n_upsets = (y_test == 0).sum()` (games where higher seed lost)
  - `upset_correct = ((y_prob < 0.5) & (y_test == 0)).sum()` (upsets the model predicted)
- Collect all per-year results into a list of dicts.
- Also collect all y_true and y_prob across folds (for calibration).

**Print a formatted comparison table:**

**Function `check_calibration(y_true_all, y_prob_all, save_path='models/calibration_curve.png')`:**
- Uses `sklearn.calibration.CalibrationDisplay.from_predictions()` to create a reliability diagram.
- Parameters: `n_bins=10, strategy='uniform', name='Logistic Baseline'`.
- Add a reference diagonal line (already included by CalibrationDisplay).
- Title: "Calibration Curve - Logistic Baseline (2022-2025 Holdout)"
- Save to `save_path` with `dpi=150, bbox_inches='tight'`.
- Print bin-level summary: for each of the 10 bins, print the mean predicted probability and the actual fraction of positives. Use `sklearn.calibration.calibration_curve()` for this.
- Print max deviation from perfect calibration (max |predicted - actual| across bins).

**Function `check_top_seed_overconfidence(df, artifact, processed_dir='data/processed')`:**
- From the matchup dataset, filter to games where BOTH team_a_seed <= 10 AND team_b_seed <= 10 (top-10 seeded matchups — these are late-round games between strong teams).
- For each such game, compute predicted probability using the model.
- If any predicted probability exceeds 0.90, print a WARNING (don't crash — just flag it).
- Print summary: N top-seed matchups checked, max probability observed, min probability observed.
- Return True if all probabilities are < 0.90, False otherwise.
