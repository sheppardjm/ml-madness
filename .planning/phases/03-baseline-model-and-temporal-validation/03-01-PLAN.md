<objective>
Fetch historical Torvik efficiency ratings for all 22 valid tournament seasons (2003-2025, excl. 2020) and build the inline feature engineering function that computes per-matchup differential features from those ratings.

Purpose: The training dataset for the logistic regression baseline requires historical team-level efficiency metrics (adjOE, adjDE, barthag, tempo, WAB) joined to tournament game records. Only 2025 data exists currently (`current_season_stats.parquet`). This plan fills the historical gap and creates the feature computation layer that all subsequent modeling plans depend on.

- `data/processed/historical_torvik_ratings.parquet` — per-team metrics for 22 seasons
- `src/models/features.py` — `compute_features()` and `build_matchup_dataset()` functions
- `src/ingest/fetch_historical_ratings.py` — one-time historical data fetch script
</objective>

2. Create `src/models/features.py` with the following:

**Constants:**
```python
FEATURE_COLS = ['adjoe_diff', 'adjde_diff', 'barthag_diff', 'seed_diff', 'adjt_diff', 'wab_diff']
```

**Function `compute_features(season, team_a_id, team_b_id, stats_lookup)`:**
- Takes a season (int), two Kaggle team IDs, and a `stats_lookup` dict keyed by `(season, team_id)` -> dict of stats.
- Computes differential features: `team_a_stat - team_b_stat` for each of adjoe, adjde, barthag, seed, adjt, wab.
- Returns a dict with keys matching `FEATURE_COLS`.
- If either team is missing from stats_lookup, raise `KeyError` with a descriptive message.
- **adjde_diff note**: Lower adj_d is better (fewer points allowed). The differential still works — a positive adjde_diff means team_a has a *higher* (worse) defensive rating. The model will learn the correct sign.

**Function `build_stats_lookup(processed_dir='data/processed')`:**
- Reads `historical_torvik_ratings.parquet` and `seeds.parquet` via DuckDB.
- Builds and returns a dict: `{(season, kaggle_team_id): {'adj_o': float, 'adj_d': float, 'barthag': float, 'adj_t': float, 'wab': float, 'seed_num': int}}`.
- For 2025 season specifically, also read from `current_season_stats.parquet` (since historical_torvik may use archive data; current_season_stats was the dedicated Phase 2 output).
- The seed_num comes from seeds.parquet (only tournament teams have seeds; non-tournament teams get no entry).
- Return the lookup dict.

**Function `build_matchup_dataset(processed_dir='data/processed')`:**
- Uses DuckDB SQL to join `tournament_games.parquet` with `seeds.parquet` (for both winner and loser seeds).
- Filters: `WHERE NOT IsFirstFour AND ws.SeedNum != ls.SeedNum` (exclude First Four and equal-seed games).
- Canonical ordering: team_a = team with lower SeedNum (better seed). If equal SeedNum (shouldn't happen after filter), use lower TeamID as team_a.
- Label: 1 if team_a (lower SeedNum) won, 0 otherwise.
- Call `build_stats_lookup()` to get the stats dict.
- For each game row, call `compute_features()` to get the 6 differential features.
- Add features as columns alongside Season, team_a_id, team_b_id, team_a_seed, team_b_seed, label.
- Drop any rows where features are NaN (teams without historical stats — log a warning with count).
- Return the DataFrame.
- Print summary: total matchups, matchups per season, NaN-dropped rows, label distribution.

3. Add a `__main__` block that calls `build_matchup_dataset()`, prints shape and first few rows, and verifies:
   - No NaN values in FEATURE_COLS
   - Label is binary (0 or 1)
   - Season column spans multiple years
   - Feature signs make sense (print mean of each feature — adjoe_diff should be positive on average since higher seeds tend to have better offense)

4. Run: `uv run python -m src.models.features`
