  <name>Task 1: Create VIF analysis module</name>
  <files>src/models/vif_analysis.py</files>
  <action>
    Create `src/models/vif_analysis.py` with:

    1. `compute_vif(X: np.ndarray, feature_names: list[str]) -> pd.DataFrame` function:
       - Takes feature matrix X (no intercept column -- statsmodels adds its own internally) and feature name list
       - Computes VIF for each feature using `statsmodels.stats.outliers_influence.variance_inflation_factor`
       - Returns DataFrame with columns: 'feature', 'vif', 'status'
       - Status logic: VIF > 10 = "EXCEEDS_THRESHOLD", VIF > 5 = "BORDERLINE", else "OK"
       - Sorted by VIF descending

    2. `run_vif_analysis(processed_dir="data/processed") -> dict` function:
       - Calls `build_matchup_dataset(processed_dir)` to get the full feature matrix
       - Extracts `df[FEATURE_COLS].values` as X (numpy array, no intercept)
       - Calls `compute_vif(X, FEATURE_COLS)`
       - Prints a formatted table with feature name, VIF value, status, and threshold indicator
       - Builds a report dict with:
         - "analysis_date": today's ISO date
         - "n_samples": number of matchups
         - "n_features": 6
         - "threshold": 10
         - "features": list of dicts with {name, vif, status}
         - "exceeds_threshold": list of feature names with VIF > 10
         - "decision": "KEEP_ALL" (if barthag_diff exceeds but we keep it)
         - "decision_rationale": A clear explanation: "barthag_diff VIF exceeds 10 (multicollinearity with adjoe_diff and adjde_diff). This is expected and documented since Phase 3 decision [03-01]. Regularized logistic regression (L2 penalty) and gradient boosting (XGBoost, LightGBM) are robust to moderate multicollinearity. Dropping barthag_diff would require retraining all existing models. The feature is retained as it contributes predictive signal via the ensemble meta-learner (coefficient 1.70 in TwoTierEnsemble)."
         - "without_barthag": dict with VIF values if barthag_diff were dropped (for documentation)
       - Returns the report dict

    3. `__main__` block:
       - Calls `run_vif_analysis()`
       - Writes the report dict to `models/vif_report.json` using `json.dump(report, indent=2)`
       - Prints "VIF report written to models/vif_report.json"

    IMPORTANT: Do NOT add an intercept column to X. `variance_inflation_factor(exog, exog_idx)` handles intercept internally. Verify by checking that adjt_diff VIF is approximately 1.05 (it's nearly independent of other features).

    Also compute the "without_barthag" scenario: drop barthag_diff column from X, recompute VIF for remaining 5 features, and include as a sub-dict in the report showing what VIF values would be.
  </action>
