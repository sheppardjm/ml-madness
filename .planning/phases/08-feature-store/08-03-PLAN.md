  <action>
    Create `tests/test_features.py` with the following test functions:

    **SC-1: Known historical matchup fixtures**

    1. `test_compute_features_returns_all_keys(stats_lookup)`:
       - Call `compute_features("Duke", "Michigan", 2025, stats_lookup=stats_lookup)`
       - Assert return dict has exactly 6 keys matching FEATURE_COLS
       - Assert all values are float (not None or NaN)

    2. Parametrized test `test_known_matchup_feature_signs`:
       - Use `@pytest.mark.parametrize` with known matchup fixtures:
         - ("Duke", "Michigan", 2025, "adjoe_diff", "positive") -- Duke has better offense
         - ("Duke", "Michigan", 2025, "seed_diff", "negative") -- Duke has lower seed number (1 vs 5)
         - ("Duke", "Michigan", 2025, "barthag_diff", "positive") -- Duke has higher power rating
         - ("Duke", "Michigan", 2025, "adjde_diff", "negative") -- Duke allows fewer points (lower adj_d = better)
       - For each: call compute_features, check the sign of the specified feature

    3. `test_known_matchup_approximate_values(stats_lookup)`:
       - Call `compute_features("Duke", "Michigan", 2025, stats_lookup=stats_lookup)`
       - Assert `seed_diff == -4` (Duke seed 1, Michigan seed 5, exact integer)
       - Assert `adjoe_diff` is in range (10, 20) -- Duke's offense is significantly better
       - Assert `barthag_diff` is in range (0.03, 0.15) -- Duke has higher barthag

    4. `test_name_based_matches_id_based(stats_lookup)`:
       - Call `compute_features("Duke", "Michigan", 2025, stats_lookup=stats_lookup)` (name-based)
       - Call `_compute_features_by_id(2025, 1181, 1276, stats_lookup)` (ID-based, Duke=1181, Michigan=1276)
       - Assert all 6 feature values are identical

    **SC-3: Cutoff-date enforcement**

    5. `test_cutoff_enforcement_by_construction(stats_lookup)`:
       - Import `SELECTION_SUNDAY_DATES` from `src.utils.cutoff_dates`
       - For season 2025: assert `(2025, 1181)` is in stats_lookup (Duke exists)
       - Assert that stats_lookup was built from data files that predate Selection Sunday
       - The assertion is about data provenance: the `historical_torvik_ratings.parquet` is built from cbbdata archive snapshots at or before Selection Sunday dates. Assert that the stats_lookup contains entries for backtest years (2022-2025) with valid numeric values, confirming the data pipeline produced pre-SS snapshots.
       - Verify: for a known 2025 tournament team (Duke, ID 1181), assert all stat values are finite floats (not NaN, not inf)

    6. `test_stats_lookup_covers_backtest_years(stats_lookup)`:
       - For each year in BACKTEST_YEARS = [2022, 2023, 2024, 2025]:
         - Assert at least 60 teams exist in stats_lookup for that season (tournament has 64-68 teams)
         - This confirms the stats_lookup has pre-SS data for each backtest year

    **SC-4: Perspective symmetry**

    7. `test_perspective_symmetry(stats_lookup)`:
       - Call `compute_features("Duke", "Michigan", 2025, stats_lookup=stats_lookup)` -> feats_ab
       - Call `compute_features("Michigan", "Duke", 2025, stats_lookup=stats_lookup)` -> feats_ba
       - For each feature in FEATURE_COLS: assert `abs(feats_ab[k] + feats_ba[k]) < 1e-10`
       - This confirms that A-B differentials exactly negate to B-A

    8. `test_perspective_symmetry_multiple_pairs(stats_lookup)`:
       - Test symmetry for 3 different team pairs across different seasons:
         - ("Duke", "Michigan", 2025)
         - ("Gonzaga", "UCLA", 2024)  -- use canonical_name from team_normalization
         - ("Houston", "Alabama", 2023)
       - Same assertion: feats_ab[k] + feats_ba[k] == 0 for all k

    **Error handling**

    9. `test_unknown_team_raises_valueerror()`:
       - `compute_features("Totally Fake University", "Duke", 2025)` should raise `ValueError`
       - Use `pytest.raises(ValueError, match="not found")`

    10. `test_missing_season_raises_keyerror(stats_lookup)`:
        - `compute_features("Duke", "Michigan", 2002, stats_lookup=stats_lookup)` should raise `KeyError`

For other teams, use names from the kaggle_name or canonical_name columns. "Gonzaga" (1211), "UCLA" (1437), "Houston" (1220), "Alabama" (1104) are all safe canonical names.
