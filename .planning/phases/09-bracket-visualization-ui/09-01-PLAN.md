  <action>
Create `src/ui/__init__.py` (empty file to make ui a package).

Create `src/ui/data_loader.py` with these functions:

1. `load_model()` -- decorated with `@st.cache_resource`:
   - Reads `models/selected.json` (json.load)
   - Loads the artifact via `joblib.load(meta["model_artifact_path"])`
   - Returns tuple `(ensemble_artifact, meta)` where ensemble_artifact is the full joblib dict (contains 'ensemble' key with TwoTierEnsemble, 'scaler' key with StandardScaler, 'feature_names' key with FEATURE_COLS)
   - meta is the selected.json dict

2. `build_ensemble_predict_fn(_artifact, season=2025)` -- decorated with `@st.cache_resource`:
   - Extracts `ensemble = _artifact['ensemble']` (TwoTierEnsemble instance)
   - Extracts `scaler = ensemble.scaler` (the StandardScaler stored on the ensemble)
   - Calls `build_stats_lookup("data/processed")` from src.models.features
   - Imports `_compute_features_by_id` and `FEATURE_COLS` from src.models.features
   - Returns a closure `predict_fn(team_a_id: int, team_b_id: int) -> float` that:
     a. Calls `_compute_features_by_id(season, team_a_id, team_b_id, stats_lookup)`
     b. Extracts feature values in FEATURE_COLS order into np.array shape (1, 6)
     c. Applies `scaler.transform(x)` to scale features
     d. Calls `ensemble.predict_proba(x_scaled)` to get probabilities
     e. Returns `float(np.clip(prob[0, 1], 0.0, 1.0))`
   - IMPORTANT: TwoTierEnsemble.predict_proba() expects ALREADY-SCALED features per decision [06-03]. The scaler.transform() happens inside this predict_fn, NOT inside predict_proba().
   - Uses underscore-prefix `_artifact` to prevent Streamlit from trying to hash the unhashable joblib artifact.

3. `run_deterministic(_predict_fn, _seedings, season=2025)` -- decorated with `@st.cache_data`:
   - Calls `simulate_bracket(seedings=_seedings, predict_fn=_predict_fn, mode="deterministic", season=season)`
   - Note: no stats_lookup passed (championship_game will be None; that's fine for bracket display)
   - Returns the result dict.
   - Uses underscore-prefix `_predict_fn` and `_seedings` to prevent unhashable type errors.

4. `run_monte_carlo(_predict_fn, _seedings, season=2025, n_runs=10000, seed=42)` -- decorated with `@st.cache_data`:
   - Calls `simulate_bracket(seedings=_seedings, predict_fn=_predict_fn, mode="monte_carlo", n_runs=n_runs, seed=seed, season=season)`
   - Returns the result dict.
   - Uses underscore-prefix parameters for unhashable args.

5. `load_team_info(season=2025)` -- decorated with `@st.cache_data`:
   - Uses duckdb to query `data/processed/seeds.parquet` for the given season
   - Returns a tuple of three dicts:
     a. `team_id_to_name: dict[int, str]` -- TeamID -> TeamName
     b. `team_id_to_seed: dict[int, str]` -- TeamID -> Seed label (e.g., "W01")
     c. `team_id_to_seednum: dict[int, int]` -- TeamID -> SeedNum (integer)
   - This is the canonical team display name source per research pitfall 5.

6. `load_seedings_cached(season=2025)` -- decorated with `@st.cache_data`:
   - Calls `load_seedings(season)` from src.simulator.bracket_schema
   - Returns the seedings dict.

Imports needed:

<task type="auto">
  <name>Task 3: Create app.py entry point with sidebar and tab skeleton</name>
  <files>app.py</files>
  <action>
Create `app.py` as the Streamlit entry point. This file must follow the set_page_config-first rule strictly.

Structure:

```python
import streamlit as st

# MUST be the very first Streamlit command -- before any other st.* calls or imports that trigger st.*
st.set_page_config(
    page_title="March Madness 2026 Predictions",
    layout="wide",
    initial_sidebar_state="expanded",
)

from src.ui.data_loader import (
    build_ensemble_predict_fn,
    load_model,
    load_seedings_cached,
    load_team_info,
    run_deterministic,
    run_monte_carlo,
)

# --- Session state initialization ---
if "season" not in st.session_state:
    st.session_state.season = 2025

season = st.session_state.season

# --- Load data (all cached) ---
artifact, model_meta = load_model()
seedings = load_seedings_cached(season)
predict_fn = build_ensemble_predict_fn(artifact, season)
team_id_to_name, team_id_to_seed, team_id_to_seednum = load_team_info(season)

# --- Run simulations (cached) ---
det_result = run_deterministic(predict_fn, seedings, season)
mc_result = run_monte_carlo(predict_fn, seedings, season)

# --- Sidebar ---
with st.sidebar:
    st.header("Model Info")
    st.caption(f"Model: {model_meta['selected_model']}")
    st.caption(f"Type: {model_meta['model_type']}")
    st.caption(f"Mean Brier: {model_meta['mean_brier']:.4f}")
    st.caption(f"Season: {season}")
    st.divider()
    champion = det_result["champion"]
    champ_name = team_id_to_name.get(champion["team_id"], str(champion["team_id"]))
    st.metric("Predicted Champion", champ_name)
    st.caption(f"Championship win prob: {champion['win_prob']:.1%}")

    mc_champ = mc_result["champion"]
    mc_champ_name = team_id_to_name.get(mc_champ["team_id"], str(mc_champ["team_id"]))
    st.metric("MC Champion", mc_champ_name)
    st.caption(f"Confidence: {mc_champ['confidence']:.1%} (10K runs)")

# --- Main content with tabs ---
st.title("March Madness 2026 Bracket Predictions")

tab_bracket, tab_advancement, tab_champion = st.tabs(
    ["Bracket", "Advancement Probabilities", "Champion"]
)

with tab_bracket:
    st.info("Bracket visualization will be rendered here (Plan 09-03)")

with tab_advancement:
    st.info("Advancement probability table will be rendered here (Plan 09-04)")

with tab_champion:
    st.info("Champion panel will be rendered here (Plan 09-04)")
```

Key rules followed:
- `st.set_page_config()` is the absolute first Streamlit command (line 3-7, before any imports that might trigger st.*)
- All data loading functions are decorated with @st.cache_resource or @st.cache_data in data_loader.py, so reruns are instant
- Unhashable args (predict_fn, seedings, artifact) use underscore-prefix convention
- layout="wide" is mandatory for 68-team bracket display
- Tab placeholders will be filled by plans 09-03 and 09-04
