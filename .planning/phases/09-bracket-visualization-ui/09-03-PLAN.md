Create `src/ui/bracket_svg.py` that builds a complete SVG string from layout coordinates and simulation results.

**Function: `render_bracket_svg_string(det_result, layout, team_id_to_name, team_id_to_seednum, season=2025) -> str`**

Parameters:
- `det_result`: Dict from `simulate_bracket(mode="deterministic")` -- has `slots` dict and `champion` dict
- `layout`: Dict from `compute_bracket_layout()` -- has `slots`, `connectors`, `canvas_width`, `canvas_height`
- `team_id_to_name`: Dict mapping team_id -> display name string
- `team_id_to_seednum`: Dict mapping team_id -> integer seed number
- `season`: Tournament season year

**IMPORTANT -- internal data resolution:** This function obtains `slot_tree` and `seedings` internally. It must NOT require them as parameters. At the top of the function body, call:
```python
slot_tree = build_slot_tree(season)
seedings = load_seedings(season)
```
These are needed by `_resolve_slot_teams()` to map StrongSeed/WeakSeed references to actual team IDs. Add the following imports at the top of the module:
```python
from src.simulator.bracket_schema import build_slot_tree, load_seedings
```

Returns: Complete SVG string (not HTML -- the caller wraps in HTML).

**SVG structure:**

1. SVG root element with `viewBox="0 0 {canvas_width} {canvas_height}"` and explicit width/height attributes. Use a dark background color (#0e1117) to match Streamlit dark theme.

2. For each slot in layout["slots"]:
   a. Get the simulation result from `det_result["slots"][slot_id]` -- contains `team_id`, `win_prob`, `round`
   b. Look up the two teams competing in this slot:
      - For the WINNER: use `det_result["slots"][slot_id]["team_id"]` and look up name/seed
      - For the slot's two INPUT teams: call `_resolve_slot_teams(slot_id, slot_tree, seedings, det_result)` which resolves StrongSeed and WeakSeed references. If a reference is a seed label (e.g., "W01"), look it up in `seedings` to get the team_id. If a reference is another slot_id (e.g., "R2W1"), look up the winner from `det_result["slots"]`.
   c. Draw a game box at (x, y, w, h) from the layout:
      - Background rectangle with rounded corners, fill color based on whether this is the champion slot (#1e5a3a for champion, #1a1a2e for normal), stroke #4a90d9
      - Two rows inside the box:
        - Top row: team A (StrongSeed's resolved team) -- show "(seed) TeamName  prob%" with the winner's text bold/bright and the loser's text dimmed
        - Bottom row: team B (WeakSeed's resolved team) -- same format
      - The winner row gets highlighted text (#ffffff) and the loser row gets dimmed (#888888)
      - Win probability shown on the winner's row, right-aligned, in green (#4ec9b0)
   d. For right-side regions (Y, Z -- where `side == "right"`): text is still left-aligned within the box (SVG text is always left-to-right), but the box x-position is already mirrored by the layout module.

3. For each connector in layout["connectors"]:
   - Draw a polyline or path connecting the points with stroke #4a90d9, stroke-width 1, fill none
   - Use L-shaped connectors: horizontal from child output -> vertical -> horizontal into parent input

4. Region labels: Add text labels for each region name ("WEST", "EAST", "SOUTH", "MIDWEST" -- mapped from W, X, Y, Z) positioned above each region's R1 column.

5. Center column labels: "FINAL FOUR" above R5 slots, "CHAMPIONSHIP" above R6CH.

**Helper function: `_resolve_slot_teams(slot_id, slot_tree, seedings, det_result) -> tuple[int, int]`**
Returns (strong_team_id, weak_team_id) for a given slot by resolving the StrongSeed and WeakSeed references through the slot tree and simulation results.
- `slot_tree`: Dict from `build_slot_tree(season)` -- maps slot_id to slot info including StrongSeed/WeakSeed
- `seedings`: Dict from `load_seedings(season)` -- maps seed label (e.g., "W01") to team_id
- If StrongSeed/WeakSeed is a seed label (present in `seedings`), return `seedings[label]`
- If StrongSeed/WeakSeed is another slot_id (present in `det_result["slots"]`), return `det_result["slots"][ref]["team_id"]`

**Helper function: `_svg_game_box(x, y, w, h, strong_name, strong_seed, weak_name, weak_seed, winner_id, strong_id, weak_id, win_prob, is_champion, side) -> str`**
Returns an SVG `<g>` group with the game box rectangle and text elements.

**Font:** Use `font-family="Consolas, Monaco, monospace"` for consistent character widths. Font size 9px for team names, 8px for seed labels and probabilities. Truncate team names longer than 14 characters with "..." to prevent overflow.

**Color scheme (dark theme):**
- Background: #0e1117
- Normal box fill: #1a1a2e
- Champion box fill: #1e5a3a (green tint)
- Box stroke: #4a90d9
- Winner text: #ffffff
- Loser text: #666666
- Probability text: #4ec9b0
- Connector lines: #3a6a9a
- Region labels: #888888
