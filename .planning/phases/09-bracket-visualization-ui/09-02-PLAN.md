Create `src/ui/bracket_layout.py` with the following:

**Constants:**
```python
SLOT_WIDTH = 150     # pixels per game box width
SLOT_HEIGHT = 22     # pixels per game slot line (each game box has 2 lines: team A and team B)
BOX_HEIGHT = 44      # full game box height (2 * SLOT_HEIGHT)
ROUND_GAP = 30       # horizontal gap between rounds
FF_GAP = 20          # horizontal gap for First Four pre-column
REGION_V_GAP = 20    # vertical gap between two regions on the same side
CENTER_GAP = 80      # horizontal gap for the Final Four center column
CONNECTOR_OFFSET = 5 # horizontal line extension for connectors
```

**Layout structure (4-panel + center):**
The bracket is displayed as:
```
[FF_W] [R1_W ... R4_W]  [R1_X ... R4_X]  [R5WX]  [R5YZ]  [R4_Y ... R1_Y]  [R4_Z ... R1_Z] [FF_Z]
                                            [R6CH]
```

Left side: W region (R1 leftmost, R4 rightmost) stacked above X region (R1 leftmost, R4 rightmost).
Right side: Y region (R4 leftmost, R1 rightmost -- mirrored) stacked above Z region (same mirroring).
Center: R5WX, R5YZ, R6CH.

**Function: `compute_bracket_layout(season: int = 2025) -> dict`**

Returns:
```python
{
    "slots": {
        slot_id: {
            "x": int,      # left edge of game box
            "y": int,      # top edge of game box
            "w": int,      # width (SLOT_WIDTH)
            "h": int,      # height (BOX_HEIGHT)
            "round": int,  # round number (0=FF, 1-6)
            "region": str,  # "W", "X", "Y", "Z", "FF" (Final Four/Championship)
            "side": str,    # "left" or "right" (for text alignment mirroring)
        }
    },
    "connectors": [
        {
            "from_slot": str,   # child slot_id
            "to_slot": str,     # parent slot_id
            "points": [(x1,y1), (x2,y2), (x3,y3)],  # L-shaped line points
        }
    ],
    "canvas_width": int,
    "canvas_height": int,
}
```

**Coordinate computation logic:**

For each region (W, X, Y, Z):
- Region has slots R1 (8 games), R2 (4 games), R3 (2 games), R4 (1 game = Elite 8)
- Within a round, slots are spaced with increasing vertical gaps:
  - R1: 8 game boxes spaced at `BOX_HEIGHT + 4` pixels apart (tight)
  - R2: 4 game boxes, each centered between the two R1 games that feed into it
  - R3: 2 game boxes, each centered between the two R2 games that feed into it
  - R4: 1 game box, centered between the two R3 games that feed into it

Vertical position formula for round `r` (1-4), slot index `s` (0-based within round):
```python
base_spacing = BOX_HEIGHT + 4  # R1 spacing
spacing = base_spacing * (2 ** (r - 1))
y = s * spacing + (spacing - BOX_HEIGHT) // 2
```

For left-side regions (W on top, X on bottom):
- x for round r: `FF_column_width + (r - 1) * (SLOT_WIDTH + ROUND_GAP)`
- W region starts at y=0
- X region starts at y = (W region height + REGION_V_GAP)

For right-side regions (Y on top, Z on bottom) -- MIRRORED:
- x for round r: `canvas_width - FF_column_width - r * (SLOT_WIDTH + ROUND_GAP) + ROUND_GAP` (mirrored so R1 is rightmost)
- Y region starts at y=0
- Z region starts at y = (Y region height + REGION_V_GAP)

For Final Four + Championship (center column):
- R5WX: x at center, y centered between W R4 and X R4 outputs
- R5YZ: x at center, y centered between Y R4 and Z R4 outputs
- R6CH: x at center, y centered between R5WX and R5YZ

**First Four slots:**
- FF slots (W16, X11, Y11, Y16) appear as a narrow pre-column
- Left-side FF slots: to the left of R1, vertically aligned with the R1 slot they feed into
- Right-side FF slots: to the right of R1 (mirrored side), vertically aligned

To determine which R1 slot a FF slot feeds into, use `build_slot_tree()` and find R1 slots whose StrongSeed or WeakSeed matches the FF slot_id.

**Connector lines:**
For each slot that has child slots (StrongSeed or WeakSeed referencing another slot_id, not a seed label):
- Draw an L-shaped connector from the child slot's output edge to the parent slot's input edge
- Left-side connectors: horizontal line right from child, then vertical to parent's y center, then horizontal into parent
- Right-side connectors: horizontal line left from child, then vertical, then horizontal into parent
