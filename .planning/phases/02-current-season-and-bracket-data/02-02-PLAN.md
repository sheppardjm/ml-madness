    1. Create `src/ingest/fetch_bracket.py` with the following functions:

       **Constants:**
       ```python
       ESPN_SCOREBOARD_URL = (
           "https://site.api.espn.com/apis/site/v2/sports/basketball/"
           "mens-college-basketball/scoreboard"
       )
       # 2026 bracket dates: First Four (March 17-18), Round of 64 (March 19-20)
       BRACKET_DATES_2026 = ["20260317", "20260318", "20260319", "20260320"]
       ```

       **`fetch_espn_bracket(dates: list[str] = BRACKET_DATES_2026) -> pd.DataFrame`**
       - For each date in dates, GET ESPN scoreboard with params `{"dates": date, "groups": 100, "limit": 200}`
       - Use requests.Session with HTTPAdapter(max_retries=Retry(total=3, backoff_factor=1)) for resilience
       - timeout=30 per request
       - For each event -> competition -> competitor:
         - team_espn_name: `competitor["team"]["displayName"]`
         - espn_team_id: `str(competitor["team"]["id"])`
         - seed: `competitor["curatedRank"]["current"]` (integer 1-16)
         - region: parse from `competition["notes"][]["headline"]` using regex `r"- ([A-Za-z ]+) Region -"`
       - Deduplicate by espn_team_id (teams appear in multiple games across dates)
       - Return DataFrame with columns: team_espn_name, espn_team_id, seed, region
       - If the result has 0 rows, print a clear message: "No tournament games found. ESPN bracket data is only available after Selection Sunday (March 15, 2026)."
       - Do NOT raise an exception on 0 rows — that's expected before Selection Sunday

       **IMPORTANT ESPN API pitfalls (from RESEARCH.md):**
       - Seed is `curatedRank.current`, NOT a dedicated "seed" field
       - Region requires regex parsing from notes headline
       - curatedRank may be missing for non-tournament games — check for None
       - groups=100 is required to get all D1 games (not just top conferences)

       **`load_bracket_csv(csv_path: str = "data/seeds/bracket_manual.csv") -> pd.DataFrame`**
       - Read CSV with dtype: `{"espn_team_id": str, "seed": int, "team_espn_name": str, "region": str}`
       - Validate: exactly 68 rows
       - Validate: exactly 4 unique regions, each a valid region name (East, West, Midwest, South)
       - Validate: seeds 1-16 present in each region (4 regions x 16 seeds = 64 unique team-seed-region combos, plus 4 play-in teams with duplicate seeds 11/12/16)
       - Return DataFrame with same columns as fetch_espn_bracket: team_espn_name, espn_team_id, seed, region

       **`load_bracket(csv_path: str = "data/seeds/bracket_manual.csv", auto_fetch: bool = True, dates: list[str] = BRACKET_DATES_2026) -> pd.DataFrame`**
       - If auto_fetch is True:
         - Try fetch_espn_bracket(dates)
         - If result has exactly 68 rows, return it
         - If fewer than 68 rows, print warning and fall through to CSV
       - On any exception from fetch_espn_bracket, print the error and fall through to CSV
       - Load CSV via load_bracket_csv(csv_path)
       - Return the CSV DataFrame
       - The caller should NOT need to know which source was used — the schema is identical

       **`resolve_bracket_teams(bracket_df: pd.DataFrame, processed_dir: str = "data/processed") -> pd.DataFrame`**
       - Takes a bracket DataFrame (from either source) and resolves each team to the normalization table
       - Join strategy:
         a. Match bracket.espn_team_id -> normalization.espn_slug (if espn_slug was populated in 02-01)
         b. For unmatched: fuzzy match bracket.team_espn_name -> normalization.espn_name or canonical_name
         c. For still unmatched: log warning with team name
       - Adds columns: kaggle_team_id, canonical_name
       - Returns enriched DataFrame with original columns + kaggle_team_id + canonical_name
       - Asserts all 68 teams resolved (raises if any unresolved)
