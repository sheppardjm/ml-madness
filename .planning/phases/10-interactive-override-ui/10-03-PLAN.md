1. **Fixture: predict_fn and seedings** (session-scoped for performance):
   ```python
   from src.simulator.bracket_schema import build_predict_fn, load_seedings
   from src.simulator.simulate import simulate_bracket

   @pytest.fixture(scope="session")
   def sim_context():
       predict_fn, stats = build_predict_fn(season=2025)
       seedings = load_seedings(2025)
       return predict_fn, seedings
   ```

2. **test_no_override_baseline(sim_context)**: Run simulate_bracket with override_map=None. Verify all 67 slots have team_id and win_prob. Store champion team_id. This is the control.

3. **test_r1_override_cascades_downstream(sim_context)**:
   - Run baseline (no overrides) and record the winner of slot R1W1.
   - Find the LOSER of R1W1 (the team that didn't win in baseline).
   - Override R1W1 to the loser: `override_map={"R1W1": loser_id}`.
   - Run simulation with override.
   - Assert R1W1 winner is now loser_id.
   - Assert R1W1 slot has `overridden=True`.
   - Assert downstream slot R2W1 has a different winner than baseline (because one of R2W1's feeders changed). If R2W1's baseline winner was the R1W1 winner, R2W1 should now have a different winner.
   - Assert at least one slot in R3W1 or higher is also different from baseline (cascade propagated).

4. **test_override_does_not_affect_other_regions(sim_context)**:
   - Override R1W1 as above.
   - Assert slots R1X1 through R1Z8 are identical to baseline (other regions unaffected).
   - Assert R4X1, R4Y1, R4Z1 unchanged.

5. **test_empty_override_equals_no_override(sim_context)**:
   - Run with override_map={} and with override_map=None.
   - Assert both results have identical champion team_id and all slot winners match.

6. **test_override_marks_slot_overridden(sim_context)**:
   - Override R1W1 to the underdog.
   - Assert `result["slots"]["R1W1"]["overridden"]` is True.
   - Assert `result["slots"]["R1W2"]["overridden"]` is False (not overridden).

7. **test_mc_override_changes_advancement_probs(sim_context)**:
   - Run MC baseline (n_runs=1000, seed=42, no overrides).
   - Run MC with override_map={"R1W1": underdog_id} (n_runs=1000, seed=42).
   - Assert that the overridden underdog team's Round of 32 advancement probability increased (should be 1.0 since it's forced to win R1).
   - Assert champion confidence may differ from baseline.

Use `mode="deterministic"` for tests 2-6 (fast, ~0.01s each). Use `mode="monte_carlo"` with n_runs=1000 for test 7 (fast enough at ~0.02s).

To find the "underdog" for a slot, run baseline and find which team LOST in slot R1W1 by comparing the two teams that competed there. Use `build_slot_tree()` to find StrongSeed and WeakSeed references for R1W1, resolve them to team_ids via seedings, and pick the one that is NOT the winner.
