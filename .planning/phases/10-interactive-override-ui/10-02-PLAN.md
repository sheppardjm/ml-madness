  <action>
Create new file `src/ui/override_controls.py` with the following components:

1. **OVERRIDE_ROUNDS constant**: A list of (round_display_name, [slot_ids]) tuples covering all 67 game slots including First Four:
   ```python
   OVERRIDE_ROUNDS = [
       ("First Four", ["W16", "X11", "Y11", "Y16"]),
       ("Round of 64", [f"R1{r}{i}" for r in "WXYZ" for i in range(1, 9)]),
       ("Round of 32", [f"R2{r}{i}" for r in "WXYZ" for i in range(1, 5)]),
       ("Sweet 16", [f"R3{r}{i}" for r in "WXYZ" for i in range(1, 3)]),
       ("Elite 8", [f"R4{r}1" for r in "WXYZ"]),
       ("Final Four", ["R5WX", "R5YZ"]),
       ("Championship", ["R6CH"]),
   ]
   ```

2. **_make_override_callback(slot_id, selectbox_key)**: Returns a closure that reads the selectbox value from `st.session_state[selectbox_key]` and updates `st.session_state["override_map"]`. If selected value is None (meaning "Model pick"), remove the slot from override_map via `.pop(slot_id, None)`. If selected value is an int (team_id), set `override_map[slot_id] = team_id`.

   CRITICAL: The callback must read from `st.session_state[selectbox_key]` (the selectbox's own key), NOT from the parameter. The selectbox value is stored at `st.session_state[selectbox_key]` and is a tuple `(label, team_id_or_None)`. Extract `team_id = st.session_state[selectbox_key][1]`.

3. **_render_slot_override(slot_id, det_result, team_id_to_name, team_id_to_seednum, season)**: Renders a single selectbox for one game slot.
   - Import and call `_resolve_slot_teams(slot_id, slot_tree, seedings, det_result)` from bracket_svg to get (strong_id, weak_id).
   - Build slot_tree and seedings from `build_slot_tree(season)` and `load_seedings(season)`.
   - If either team is None (unresolvable -- e.g., depends on an earlier slot not yet filled), skip rendering this selectbox entirely.
   - Create options list: `[("Model pick", None), ("(seed) StrongName", strong_id), ("(seed) WeakName", weak_id)]`.
   - Determine current index: look up `st.session_state["override_map"].get(slot_id)` and find matching option index. Default to 0 ("Model pick").
   - Use key=f"override_{slot_id}" for the selectbox widget key.
   - Use `format_func=lambda opt: opt[0]` to display only the label string.
   - Attach `on_change=_make_override_callback(slot_id, f"override_{slot_id}")`.
   - MAX_NAME_LEN = 16 for selectbox labels (slightly longer than SVG's 14 since selectbox has more horizontal space).

4. **build_override_controls(det_result, team_id_to_name, team_id_to_seednum, season)**: Main entry point.
   - Iterates OVERRIDE_ROUNDS. For each round:
     - Count active overrides for this round: `n_active = sum(1 for sid in slot_ids if sid in st.session_state.get("override_map", {}))`.
     - Build label: `f"{round_name}"` + `f" ({n_active} override{'s' if n_active != 1 else ''})"` if n_active > 0.
     - Create `st.expander(label, expanded=False)`.
     - Inside expander, render each slot's selectbox via `_render_slot_override()`.

5. **reset_overrides()**: Standalone callback function for the reset button.
   ```python
   def reset_overrides():
       st.session_state["override_map"] = {}
   ```

Module-level imports: streamlit, bracket_schema.build_slot_tree, bracket_schema.load_seedings, bracket_svg._resolve_slot_teams.

IMPORTANT: Use deferred imports inside functions for build_slot_tree, load_seedings, and _resolve_slot_teams (same pattern as advancement_table.py per decision [09-04]) so the module is safe to import outside Streamlit context.

IMPORTANT: The selectbox `format_func` receives the actual option value (a tuple), NOT the index. So `format_func=lambda opt: opt[0]` extracts the display label from the (label, team_id) tuple.
  </action>
