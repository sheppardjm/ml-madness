# Milestone v1.0: MVP

**Status:** SHIPPED 2026-03-10
**Phases:** 1-10
**Total Plans:** 37

## Overview

This project builds a Python-first ML pipeline that ingests 20+ years of NCAA tournament data, engineers efficiency-based features, trains a stacking ensemble, simulates 10,000 bracket outcomes via Monte Carlo, and surfaces results through an interactive Streamlit bracket UI. The five architectural layers — data pipeline, feature store, model layer, bracket simulator, and web UI — are built in strict dependency order, with backtesting and ensemble model selection woven in after a validated baseline exists. The deliverable is a working bracket predictor ready before Selection Sunday 2026 (mid-March).

## Milestone Summary

**Key Decisions:**
- ClippedCalibrator [0.05, 0.89] over isotonic calibration (eliminated overconfidence with minimal Brier penalty)
- Manual OOF temporal stacking via TwoTierEnsemble (sklearn StackingClassifier incompatible with walk-forward splits)
- cbbdata 2024-25 archive as proxy for 2025-26 season (cbbdata not yet indexing current season)
- DuckDB for ingestion, Parquet for storage (fast analytical queries without server)
- Ensemble always selected as best model (Brier 0.1692 vs baseline 0.1900)

**Issues Resolved:**
- arm64 libomp incompatibility with Intel Homebrew (copied from MacPorts/Adobe)
- sklearn 1.8.0 removed cv='prefit' from CalibratedClassifierCV (custom ClippedCalibrator)
- team_aliases.csv NC A&T/NC Central ID swap, NC State/Charleston/Saint Francis false positives
- Kaggle API key malformed (manual download workaround)
- ESPN bracket data unavailable pre-Selection Sunday (CSV fallback ready)

**Issues Deferred:**
- Refresh current_season_stats.parquet when cbbdata indexes 2025-26 data
- Fetch live bracket on Selection Sunday (2026-03-15 after 6 PM ET)

**Technical Debt Incurred:**
- Phase 6: Calibration 5pp criterion fails on sparse OOF bins (10.59pp max in [0.3,0.4] with 16 samples)
- Phase 7: XGB/LGB rows missing from comparison table (Brier scores exist as constants only)
- Phase 7: No RECOMMENDED marker in table (communicated via CLI line only)
- Phase 7: XGB/LGB model_artifact_path placeholder (never triggered since ensemble always wins)
- Phase 9: Championship game score not wired to UI (stats_lookup not passed to simulate_bracket in data_loader.py)
